# 02. エージェント能力評価

> **目的**: AI エージェントの能力そのものを多角的に評価するフォーマット。特定のフェーズに紐づかず、案件全体を通じてエージェント（ツール）の能力を評価する。異なるツール間の比較のベースラインとなる。
>
> **記入タイミング**: 案件完了時
> **記入者**: 検証実行者

---

## 統一評価スケール（5 段階）

| スコア | ラベル | 定義 |
|:-----:|:-------|:-----|
| 5 | 優秀 | 人間の成果物と同等以上。修正不要 |
| 4 | 良好 | 軽微な修正（5 分以内）で利用可能 |
| 3 | 合格 | 修正が必要だが方向性は正しい（30 分以内で修正可能） |
| 2 | 不合格 | 大幅な修正が必要（やり直しに近い） |
| 1 | 失敗 | 全くの的外れ、または出力不能 |
| N/A | 該当なし | 評価対象外 |

## セクション 1: コア能力スコア（レーダーチャート用 -- 8 軸）

以下の 8 つの能力軸について、それぞれ 5 段階で評価する。レーダーチャートで可視化することを前提に、全軸を同じスケールで評価する。

| # | 能力軸 | 評価観点 | スコア (1-5) | 根拠（具体的なエビデンス） |
|:-:|:------|:---------|:----------:|:------------------------|
| 1 | **指示追従性** | 与えられた指示・制約を正確に守れるか。出力フォーマット、命名規約、禁止事項の遵守度 | | |
| 2 | **コンテキスト理解** | 既存コードベース、ドキュメント、前フェーズの成果物を正しく理解・活用できるか | | |
| 3 | **コード生成品質** | 生成されたコードの正確性、可読性、ベストプラクティス準拠度、セキュリティ考慮 | | |
| 4 | **ドキュメント生成品質** | 要件定義書、設計書、テスト計画等のドキュメントの構造・内容の品質 | | |
| 5 | **エラー回復力** | コンパイルエラー、テスト失敗、API エラー等に遭遇した際の自律的な修正能力 | | |
| 6 | **ドメイン知識活用** | 技術ドメイン固有の知識（言語仕様、フレームワーク制約、ベストプラクティス）の適用精度 | | |
| 7 | **効率性** | トークン消費、所要時間、不要な反復の少なさ。最小限の手順で正しい結果に到達するか | | |
| 8 | **協調性** | 人間のフィードバックを正しく反映するか。差し戻し指示に対して適切に修正できるか | | |

### スコアサマリ

| 指標 | 値 |
|:-----|:---|
| 8 軸合計スコア | /40 |
| 8 軸平均スコア | /5 |
