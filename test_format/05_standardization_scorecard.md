# 05. 標準化判定スコアカード

> **目的**: 複数回の検証結果を集約し、フレームワークを組織の「標準」として確定してよいかを判断するためのスコアカード。汎用コア部分とドメイン拡張部分の両方を評価対象とする。
>
> **記入タイミング**: 3 回以上の検証完了後
> **記入者**: プロジェクトリーダー

---

## 統一評価スケール（5 段階）

| スコア | ラベル | 定義 |
|:-----:|:-------|:-----|
| 5 | 優秀 | 人間の成果物と同等以上。修正不要 |
| 4 | 良好 | 軽微な修正（5 分以内）で利用可能 |
| 3 | 合格 | 修正が必要だが方向性は正しい（30 分以内で修正可能） |
| 2 | 不合格 | 大幅な修正が必要（やり直しに近い） |
| 1 | 失敗 | 全くの的外れ、または出力不能 |
| N/A | 該当なし | 評価対象外 |

## 成熟度 4 段階（標準化判定用）

| レベル | ラベル | 定義 |
|:------:|:-------|:-----|
| L1 | Initial（初期） | まだ安定していない |
| L2 | Developing（発展中） | 改善傾向にあるが変更が必要 |
| L3 | Defined（確立） | 安定して機能、軽微な調整のみ |
| L4 | Optimized（最適化） | 十分に機能、改善余地が小さい |

## セクション 7: 標準化判定

### 7.1 判定基準

| 判定 | 条件 |
|:-----|:-----|
| **標準化可** | 全メトリクス達成率 80% 以上 **かつ** コア構成要素が全て L3 以上 **かつ** 残課題の優先度「高」が 0 件 |
| **条件付き標準化可** | 全メトリクス達成率 60% 以上 **かつ** 残課題の優先度「高」が 3 件以下で対応方針が明確 |
| **標準化不可** | 上記いずれも満たさない |
