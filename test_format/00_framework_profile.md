# 00. フレームワークプロファイルシート

> **目的**: 評価対象の AI エージェントフレームワークの構成を記録するプロファイル。検証の初回に 1 回だけ記入し、以降の評価フォーマットから参照する。
>
> **記入タイミング**: 検証開始前（初回のみ。構成変更時は更新し、変更履歴を末尾に記録する）
> **記入者**: フレームワーク構築者

---

## 共通定義（全フォーマット共通）

### 統一評価スケール（5 段階）

| スコア | ラベル | 定義 |
|:-----:|:-------|:-----|
| 5 | 優秀 | 人間の成果物と同等以上。修正不要 |
| 4 | 良好 | 軽微な修正（5 分以内）で利用可能 |
| 3 | 合格 | 修正が必要だが方向性は正しい（30 分以内で修正可能） |
| 2 | 不合格 | 大幅な修正が必要（やり直しに近い） |
| 1 | 失敗 | 全くの的外れ、または出力不能 |
| N/A | 該当なし | 評価対象外 |

### 成熟度 4 段階（標準化判定用）

| レベル | ラベル | 定義 |
|:------:|:-------|:-----|
| L1 | Initial（初期） | まだ安定していない |
| L2 | Developing（発展中） | 改善傾向にあるが変更が必要 |
| L3 | Defined（確立） | 安定して機能、軽微な調整のみ |
| L4 | Optimized（最適化） | 十分に機能、改善余地が小さい |

### 汎用フェーズモデル

| 汎用フェーズ | 概要 |
|:------------|:-----|
| P-Analyze（分析） | 既存資産の把握、要件の整理 |
| P-Design（設計） | 技術設計、アーキテクチャ決定 |
| P-Build（構築） | コード生成、実装 |
| P-Verify（検証） | テスト、静的解析、品質確認 |
| P-Review（レビュー） | コードレビュー、最終確認 |

### 設計原則

1. **定量 + 定性の二層構造** -- スコアで俯瞰し、自由記述で深掘りする
2. **ツール横断の比較可能性** -- Copilot CLI、Cursor、Windsurf、Devin、Claude Code、GitHub Copilot Agent Mode 等で共通利用可能
3. **案件間の比較可能性** -- 同一スケール・同一フォーマットで異なるプロジェクトを横並び比較
4. **改善アクションへの接続** -- 問題 → 原因カテゴリ → 改善アクションを構造的にトレース可能
5. **記入負荷の最小化** -- 選択式 + 数値入力 + 最小限の自由記述
6. **拡張可能性** -- ドメイン拡張ファイルで専門領域の評価項目を追加可能
